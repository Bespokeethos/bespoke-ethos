1. Fixing the wsl usage

WSL doesnt have -admin or -root flags.
If you really want to open WSL as root, the syntax is:

powershell
# From PowerShell / CMD
wsl -u root
# or explicitly with distro, if needed:
wsl -d Ubuntu -u root

If youre fine using your normal user (which is enough for Cline, npm, etc.), just:

powershell
wsl

2. Why -workspace gives command not found

Once youre inside WSL you have:

text
conta@UR-Home:/mnt/c/Vercel$ -workspace new
-workspace: command not found

-workspace is not a program; its an option that belongs to some command.
Typing it alone makes the shell try to execute a program literally named -workspace, which obviously doesnt exist.

For Cline specifically, the extension is telling you:

Install it with: npm install -g cline, then run cline auth

So the flow should be:

bash
# Inside WSL
cd /mnt/c/Vercel   # or whatever project folder you want

# 1) Install the Cline CLI (only once)
npm install -g cline

# 2) Authenticate it
cline auth

# 3) Use it from your project directory
cline                 # interactive mode
# or e.g.
cline "Scrape the Plexus directory and save to CSV"

No bare -workspace command needed at all. Cline uses your current directory as its workspace by default. DeepWiki+1
